## 1. 프로젝트 개요

### 1.1 한 줄 정의
**numpy/pandas 수준의 기초 라이브러리만으로 AI/ML 알고리즘을 직접 구현하고, OJ 방식으로 채점받는 학습 플랫폼**

### 1.2 핵심 문제 의식

**"배웠는데 왜 실력이 안 느는가?"**

| 기존 방식 | 문제점 |
|-----------|--------|
| Kaggle / Dacon | sklearn, XGBoost 갖다 쓰면 됨. 알고리즘 구현 능력 0점이어도 상위권 가능 |
| numpy/pytorch 강좌 | 따라치기 실습. 채점도 없고 경쟁도 없음. 내재적 동기부여 결여 → 낮은 완주율 |
| CS231n 과제 | numpy-only 구현이라는 이상적인 방식이지만 스탠퍼드 재학생 전용, 외부 공개 없음 |

**공백**: numpy로 backpropagation을 직접 짜고 → 즉각 채점받고 → 순위를 겨루는 플랫폼이 존재하지 않는다.

| 카테고리 | 문제 예시 |

| **거리/유사도** | KNN, 코사인 유사도, 커널 SVM |

#### 2단계 — 딥러닝 기초 (중급 ~ 고급)

| 카테고리 | 문제 예시 |
|----------|-----------|
| **순전파 / 역전파** | 2-layer NN, backprop 직접 구현, 체인룰 적용 |

| **손실 함수** | Cross-entropy, MSE, Focal Loss 구현 |

#### 3단계 — 고급 (고급 ~ 전문가)

| 카테고리 | 문제 예시 |
|----------|-----------|
| **순환 신경망** | RNN / LSTM cell 수식 구현 |
| **어텐션** | Scaled Dot-Product Attention, Multi-head Attention |
| **자기지도 학습** | Contrastive Loss (SimCLR 스타일) |
| **강화학습 기초** | Q-learning, Policy Gradient |
| **수치 최적화** | Newton's Method, Conjugate Gradient |

### 4.4 채점 방식

**수치 비교 기반 단위 테스트** (CS231n Gradescope 방식)

```
채점 파이프라인
─────────────────────────────────────────
1. 코드 정적 분석
   - import 검사 (금지 라이브러리 감지)
   - sklearn, torch, tensorflow 등 감지 시 즉시 IE (Illegal Import)

2. 샌드박스 실행
   - Docker 격리 환경
   - 시간 제한: 문제별 (보통 5~30초)
   - 메모리 제한: 512MB

3. 다중 테스트케이스 실행
   - 예제 테스트 (공개): 제출 전 로컬 확인용
   - 채점 테스트 (비공개): 실제 점수 반영 (10~50개)
   - 엣지케이스: 빈 배열, 단일 샘플, 고차원 등

4. 수치 검증
   - np.allclose(출력, 정답, rtol=1e-4, atol=1e-6)
   - gradient 중간값 검증 (선택적)

5. 결과 반환
   AC / WA / TLE / IE (Illegal Import) / RE (Runtime Error)
```

### 4.5 채점 결과 상태

| 상태 | 의미 |
|------|------|
| `AC` | Accepted — 정답 |
| `WA` | Wrong Answer — 수치 오차 초과 또는 오답 |
| `TLE` | Time Limit Exceeded — 비효율적 구현 (루프 대신 벡터화 필요) |
| `IE` | Illegal Import — 금지 라이브러리 사용 |
| `RE` | Runtime Error — 예외 발생 |
| `MLE` | Memory Limit Exceeded |

> **TLE의 교육적 가치**: `for` 루프로 짜면 TLE, 벡터화(`@`, `np.dot`)로 짜면 AC → 행렬 연산의 중요성을 자연스럽게 체득

## 13. 미결 사항 / 검토 필요 항목

1. **scipy 허용 범위**: `scipy.linalg`, `scipy.optimize`는 허용할지? (SVD 분해 등): 잘 모르겠음. 세심한 검토가 필요함. 일단 당장은 사용하지 않는 걸로 생각하고, 이런 이슈로부터 자유로운 문제만 채점하기.
2. **부분 점수 여부**: gradient 중간값이 맞으면 부분 점수를 줄지, AC/WA만 할지: 부분점수를 주되, 100점을 맞아야지만 초록색으로 통과 표시가 달리며, 99점이더라면 노란색으로 부분점수 표시.
3. **문제 공개 정책**: AC 후 다른 사람 풀이를 볼 수 있는지: 당연히 해야 함.
4. **대회 규모**: 처음부터 실시간 대회를 넣을지, 오픈 챌린지부터 시작할지: 오픈 챌린지만 넣기.
