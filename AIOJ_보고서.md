# 1. 프로젝트 개요

## 한 줄 정의

> **numpy/pandas 수준의 기초 라이브러리만으로 AI/ML 알고리즘을 직접 구현하고, OJ 방식으로 즉시 채점받는 학습 플랫폼**

## 핵심 문제 의식

"배웠는데 왜 실력이 안 느는가?"

| 기존 방식 | 문제점 |
|-----------|--------|
| Kaggle / Dacon | sklearn, XGBoost 갖다 쓰면 됨 — 알고리즘 구현 실력 없어도 상위권 가능 |
| numpy / PyTorch 강좌 | 따라치기 실습 — 채점도 없고 경쟁도 없어 동기부여 결여, 낮은 완주율 |
| CS231n 과제 | numpy-only 구현이라는 이상적 방식이지만 스탠퍼드 재학생 전용 |

**공백**: numpy로 backpropagation을 직접 짜고 → 즉각 채점받고 → 순위를 겨루는 플랫폼이 존재하지 않는다.

# 3. 핵심 가치 및 차별화

## 4가지 핵심 가치

1. **구현이 곧 채점이다** — sklearn 한 줄이 아닌, 직접 짠 gradient descent가 통과해야 AC
2. **즉각적 피드백 루프** — 코드 제출 후 수 초 내 채점 결과 반환
3. **내재적 동기부여** — 티어, 스트릭, 대회, 랭킹으로 "다음 문제도 풀고 싶다"는 욕구 생성
4. **점진적 난이도** — 선형회귀부터 Transformer까지, 이미 아는 수학에서 시작

## 경쟁 플랫폼 대비 차별화

| 기존 플랫폼 | AIOJ |
|-------------|------|
| Kaggle: 결과 수치만 평가, 구현 무관 | 구현 코드 자체를 채점 |
| Deep-ML: OJ 스타일이지만 sklearn 허용 | numpy-only 강제, IE로 명확히 차단 |
| CS231n: 최고의 방식이지만 재학생 전용 | 누구나 접근 가능한 공개 플랫폼 |
| numpy 강좌: 동기부여 없음 | 티어·대회·스트릭으로 지속 동기 제공 |
| TLE 없는 플랫폼: 비효율 구현도 통과 | 비벡터화 구현 TLE → 행렬 연산 체득 강제 |

# 4. 문제(Problem) 시스템

## 라이브러리 정책

| 허용 | 금지 |
|------|------|
| numpy | scikit-learn |
| pandas | PyTorch |
| scipy (문제별 명시) | TensorFlow / JAX |
| matplotlib (시각화 전용, 채점 무관) | statsmodels, 기타 ML/DL 라이브러리 |
| Python 표준 라이브러리 | |

> 문제마다 허용 라이브러리를 명시 — "numpy only" / "numpy + scipy" 등으로 구분

## 문제 카테고리 (3단계 구조)

### 1단계 — ML 기초 (입문 ~ 중급)

| 카테고리 | 문제 예시 |
|----------|-----------|
| 선형 모델 | 선형회귀 (정규방정식), 경사하강법, Ridge/Lasso, 로지스틱 회귀 |
| 트리 / 앙상블 | 결정 트리 (Gini 분할), 랜덤 포레스트, 그래디언트 부스팅 |
| 클러스터링 | K-Means, DBSCAN, 계층적 클러스터링 |
| 차원 축소 | PCA (SVD 기반), t-SNE 근사 |
| 확률 모델 | Naive Bayes, GMM, EM 알고리즘 |

### 2단계 — 딥러닝 기초 (중급 ~ 고급)

| 카테고리 | 문제 예시 |
|----------|-----------|
| 순전파 / 역전파 | 2-layer NN, backprop 직접 구현, 체인룰 적용 |
| 최적화 | SGD, Momentum, AdaGrad, Adam 구현 |
| 정규화 | Batch Normalization, Dropout (순전파+역전파) |
| 합성곱 | Conv2D forward pass (im2col), MaxPooling |
| 손실 함수 | Cross-entropy, MSE, Focal Loss 구현 |

### 3단계 — 고급 (고급 ~ 전문가)

| 카테고리 | 문제 예시 |
|----------|-----------|
| 순환 신경망 | RNN / LSTM cell 수식 구현 |
| 어텐션 | Scaled Dot-Product Attention, Multi-head Attention |
| 수치 최적화 | Newton's Method, Conjugate Gradient |
| 강화학습 기초 | Q-learning, Policy Gradient |

## 채점 파이프라인

```
1. 정적 분석       → import 검사 (금지 라이브러리 즉시 IE 처리)
2. 샌드박스 실행   → Docker 격리 환경 (시간 5~30초, 메모리 512MB)
3. 테스트케이스    → 공개 예제 + 비공개 채점 테스트 (10~50개) + 엣지케이스
4. 수치 검증       → np.allclose(출력, 정답, rtol=1e-4, atol=1e-6)
5. 결과 반환       → AC / WA / TLE / IE / RE / MLE
```

## 채점 결과 상태

| 상태 | 의미 |
|------|------|
| AC | Accepted — 정답 |
| WA | Wrong Answer — 수치 오차 초과 또는 오답 |
| TLE | Time Limit Exceeded — 비벡터화 구현 (루프 → 행렬 연산 유도) |
| IE | Illegal Import — 금지 라이브러리 사용 |
| RE | Runtime Error — 예외 발생 |
| MLE | Memory Limit Exceeded |

## 부분 점수 정책

- 100점 → **초록색 AC** (완전 정답)
- 1~99점 → **노란색 부분 통과** (gradient 중간값 등 일부 정답)
- AC 후 타인 풀이 공개 (풀이 토론 해금)

# 5. 레이팅 / 티어 시스템

## 문제 난이도 (7단계, solved.ac 방식)

| 티어 | 설명 |
|------|------|
| Iron | AI 개념 이해 수준 — 수식을 코드로 옮기기 |
| Bronze | 기초 ML — 선형회귀, KNN, 정규화 |
| Silver | 중급 ML — 트리, 앙상블, 클러스터링 |
| Gold | 고급 ML + 딥러닝 기초 — backprop, CNN forward |
| Platinum | 딥러닝 심화 — LSTM, Attention, BatchNorm 역전파 |
| Diamond | 최적화 이론, 확률적 모델, 고급 RL |
| Ruby | 논문 수준 알고리즘 구현 |

## 사용자 레이팅

- 풀이한 문제 난이도의 **가중 합산**
- 쉬운 문제로는 일정 레이팅 이상 오르지 않는 구조
- **카테고리별 세부 레이팅** — 예: "최적화 Gold, 딥러닝 Silver"

## 스트릭 & 뱃지

| 항목 | 내용 |
|------|------|
| 일일 스트릭 | 하루 1문제 이상 AC 시 연속 일수 기록 |
| 카테고리 마스터 | 특정 카테고리 문제 N개 이상 AC 시 뱃지 |
| 스피드런 | 문제 최초 AC 시 특별 뱃지 |
| 대회 입상 | 정기 대회 TOP 10% 달성 시 시즌 뱃지 |

# 6. 대회(Contest) 시스템

## 대회 유형

| 유형 | 주기 | 설명 |
|------|------|------|
| 오픈 챌린지 | 상시 | 새 알고리즘 카테고리 문제 — 기간 내 자유 제출 |
| 월간 챌린지 | 월 1회 | 새 카테고리 3~5문제, 1주일간 제출 |
| 스피드 라운드 | 격주 | 2시간, 실시간 리더보드, 빠른 AC 유리 |
| 구현 마라톤 | 분기 1회 | 1주일, 복합 구현 (예: "numpy로 miniGPT 구현") |

> MVP에서는 **오픈 챌린지만** 우선 적용

## 대회 채점 특이점

- 대회 중: 예제 테스트만 공개, 숨겨진 테스트케이스로 실제 채점
- 대회 후: 전체 테스트케이스 공개 + 풀이 토론 해금
- TLE 패널티: 비벡터화 구현은 시간 제한 엄격 적용

## 4개 커리큘럼 코스

| 코스 | 기간 | 내용 |

| Transformer 밑바닥 구현 (어텐션) | 4주 | Scaled Dot-Product Attention → Multi-head → Positional Encoding → 완전한 Transformer 블록 |
